What should the UML diagram cover?

Pick the diagram type that best matches your goal

A

Class Diagram (Structure)

Shows all classes, their fields/methods, and relationships (inheritance, composition, associations). Best for understanding how the codebase is organized.

B

Sequence Diagram (Interactions)

Shows how objects communicate over time — e.g., one full simulation tick from WorldLoop through NPC update, LLM call, dialog, and result application.

C

Both — Class + Sequence (Recommended)

A class diagram for structure and a sequence diagram for the tick lifecycle. Gives a complete picture of the system in one PDF with two pages.
